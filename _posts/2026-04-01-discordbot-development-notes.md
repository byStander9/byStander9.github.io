현재 관리 중인 `discordBot` 레포를 기준으로, Discord 봇 개발 과정에서 중요했던 부분을 정리해 두려고 한다.
이 글은 완성된 소개 문서보다는 실제 개발 기록과 메모에 가깝다.

## 레포

- 레포 주소: <https://github.com/byStander9/discordBot>
- 설명: Discord Bot (Python)

## 개발하면서 중요했던 점

Discord 봇은 겉으로 보기에는 간단해 보여도, 실제로는 다음 요소들이 같이 맞아야 제대로 동작한다.

- 봇 토큰 관리
- Developer Portal 설정
- Intent 설정
- 서버 초대 권한
- 채널 접근 권한
- 이벤트 핸들링 구조
- 명령 처리 방식

특히 “코드는 맞는데 봇이 반응하지 않는” 상황은 대부분 코드보다 설정이나 권한 문제에서 나오는 경우가 많았다.

## 구조를 볼 때 확인할 것

이 레포를 다시 볼 때는 아래 순서로 확인하면 좋다.

1. 진입점 파일이 무엇인지
2. 토큰을 어떤 방식으로 관리하는지
3. 명령어/이벤트 처리가 어디에 모여 있는지
4. 기능이 파일별로 분리되어 있는지
5. 테스트 또는 실행 방법이 얼마나 단순한지

## Discord 봇 개발에서 반복해서 느끼는 점

- 토큰은 코드에 직접 넣지 않는 것이 기본
- 권한 설정이 코드만큼 중요함
- 작은 기능부터 먼저 붙이고, 동작을 확인하면서 확장하는 편이 안정적임
- 로그를 잘 남겨야 디버깅이 쉬움
- 실제 서버에서의 동작과 로컬 테스트 결과가 다를 수 있음

## 앞으로 정리하고 싶은 것

이 레포에 대해서는 나중에 아래 항목들을 더 자세히 문서화해둘 생각이다.

- 전체 파일 구조
- 주요 기능 요약
- 명령어 목록
- 실행 방법
- 배포 방법
- 겪었던 버그와 해결 방법

## 메모

Discord 봇은 단순 자동응답 수준을 넘어서기 시작하면, 설정 관리와 구조 정리가 꽤 중요해진다.
레포가 커질수록 “기능 추가”보다 “구조를 깔끔하게 유지하는 일”이 더 중요해진다는 점을 계속 느끼게 된다.
